Research Signal

Cowork は、職場AIの「実行レイヤー」が製品化され始めたというシグナル

Anthropic の Cowork と Microsoft の Copilot Cowork を中核に、Google Gemini Enterprise、Slack Agentforce、OpenAI の deep research / ChatGPT agent を重ねると、職場AIの主戦場が chat から outcome-oriented な execution layer へ移り始めていることが見えてくる。公開資料と主要論文をもとに、Cowork とは何か、どの会社がどの形で出しているのか、どのユースケースに向いているのか、そしてなぜ今この方向へ進んでいるのかを整理する。

Signal Snapshot

Cowork は chat の延長ではなく、知識労働を引き受ける execution layer として立ち上がっている

公開資料を並べると、Cowork という名称そのものを強く打ち出しているのは現時点で Anthropic と Microsoft に限られる。 ただし、その背後で起きていることはより広い。Google Gemini Enterprise、Slack の Agentforce、OpenAI の deep research / ChatGPT agent まで含めてみると、 市場は「質問に答える chat」から、「複数ステップの仕事を引き受け、成果物まで進める実行面」へ重心を移し始めている。

2社

Cowork を明示ブランド化

現時点で Cowork を前面に出しているのは Anthropic と Microsoft が中心で、他社は別名で同じ方向を製品化している。

5群

同方向の製品群が収束

Anthropic、Microsoft、Google、Slack / Salesforce、OpenAI が、context と action を束ねた workplace agent を別々の表現で打ち出している。

3層

必要なのは context、action、governance

単にモデルが賢いだけでは足りず、接続、実行、承認、監視を一体で持つことが競争軸になっている。

1結論

競争単位は UI から execution へ

これからの差は、何を答えられるかより、どこまで安全に仕事を前に進められるかで付きやすい。

What Changed

名称は限定的でも、製品の向き先はかなり共通している

重要なのは、各社が一斉に同じ名称を使っていることではない。むしろ、異なる名称のままでも、長時間実行、社内コンテキスト接続、アプリ内実行、承認と監視を備えた「仕事を進める AI」に寄っている点がシグナルである。

Anthropic Cowork

  • Cowork は Claude Code の agentic architecture を Claude Desktop に持ち込み、knowledge work 向けに広げる research preview として公開されている。
  • ローカル PC 上で plan を立て、subtask に分解し、VM で実行し、必要なら parallel workstreams を走らせ、成果物を直接ファイルへ書き戻せる。
  • 一方で audit logs、Compliance API、data exports に入らず、regulated workload には使うべきではないと公式に注意されている。

Microsoft Copilot Cowork

  • Microsoft は Copilot Cowork を Microsoft 365 の execution layer と位置づけ、Anthropic と共同で Claude Cowork を支える技術を Copilot に持ち込んでいる。
  • Work IQ で files、meetings、chats、relationships に grounded されたまま、Word、Excel、PowerPoint、Outlook、Copilot Chat の中で multi-step work を進める。
  • 加えて Agent 365 と Frontier Suite を前面に出し、identity、registry、audit、DLP、risk signals まで enterprise control plane に載せている。

Google Gemini Enterprise

  • Google Agentspace は Gemini Enterprise に統合され、「every employee, every workflow」を支える secure agentic platform として再整理されている。
  • Agentspace docs は intranet search、AI assistant、agentic workflows を一体で扱い、assistant actions と third-party connectors で Jira、Confluence、Microsoft 系 SaaS、ServiceNow などへ接続する。
  • 名称は Cowork ではないが、search、agent gallery、no-code agent designer、governance を束ねる構成はほぼ同じ方向を向いている。

Slack / Salesforce Agentforce

  • Slack の Agentforce は、agent を teammate として DM、channel mention、workflow trigger の中へ埋め込み、会話の流れのまま仕事を進める設計を取る。
  • Slack Actions と Slack Data により、canvas 作成、DM 送信、ユーザー検索、channel context 参照を使って、incident triage や account brief などの実務へ踏み込んでいる。
  • digital labor という表現は違うが、employee-facing autonomous agent を「flow of work」に置くという考え方は Cowork と同型である。

OpenAI deep research / ChatGPT agent

  • OpenAI は deep research を independently do work for you と位置づけ、ChatGPT agent では visual browser、code interpreter、apps、terminal、scheduled tasks をまとめている。
  • 複雑タスクを 5 から 30 分単位で進め、files や connected sources を使いながら、forms や spreadsheets も扱う設計は workplace execution に近い。
  • OpenAI も app controls、website blocking、workspace toggles、Compliance API 範囲などを明示し、agent を governance 対象として扱い始めている。

観測

  • 現時点で「各社が Cowork を出している」とまでは言えない。正確には、Cowork という名称は限定的で、製品方向が横展開している。
  • 共通しているのは、chat を起点にしつつも、最終的には artifact creation、workflow execution、approval-aware action へ踏み込むことだ。

Use Cases

向いているのは「何でもできる AI」ではなく、成果物と承認点が見える workflow である

公開資料に出てくるユースケースを重ねると、先に立ち上がるのは汎用知能ではない。文脈、使用ツール、完了条件、レビュー点が定義しやすい knowledge work が先行する。

1. 会議準備と営業ブリーフ

  • Google Agentspace の docs には earnings report を探して talking points を作る例があり、Slack でも customer brief や close plan の自動化が前面に出ている。
  • 社内資料、外部公開情報、過去のチャットやメールをつなぎ、meeting prep を成果物まで仕上げる用途は典型である。

2. 調査メモ、表計算、資料作成

  • Microsoft は Copilot が Word、Excel、PowerPoint の中で artifact を直接更新する点を強く打ち出している。
  • OpenAI も deep research と ChatGPT agent で web、PDF、files、spreadsheets をまたぐ調査から synthesis までを狙っており、Cowork の完成像に近い。

3. メール、予定、軽い業務調整

  • Chat-first entry point から schedule meeting、send email、calendar 参照までつなぐ流れは Microsoft と OpenAI の両方で見える。
  • この系統は、ユーザーが outcome を指定し、AI が裏で複数アプリを横断して調整する「事務処理の実行面」に向く。

4. IT / incident triage

  • Slack は Agentforce が channel context を使って service disruption を検出し、ticket を切り、現地担当へ DM を送り、incident report まで作る例を示している。
  • 人の判断が必要な確定操作は残しつつ、初動整理や情報集約を agent に任せる形は導入しやすい。

5. 社内ナレッジ検索と action の接続

  • Google の connectors、Anthropic の plugins、OpenAI の apps は、社内 knowledge を引くだけでなく、そのまま次の action へつなげる前提で設計されている。
  • 検索だけではなく、draft、update、handoff まで含めて「一つの依頼」として扱うことが Cowork 的なユースケースの核心になる。

6. 分析と定例レポート

  • Deep research、Excel / PowerPoint agents、Cowork plugins の組み合わせは、定期レポートや recurring task の自動化と相性が良い。
  • 特に、データを拾い、差分をまとめ、ドラフトを作って人間が最終承認する形は、現実の knowledge work にそのまま乗りやすい。

Why Now

この方向性は、agent research が「回答」から「完遂」へ進んできた結果として自然である

Cowork 的な製品面は突然現れたわけではない。ReAct 以降の流れを見ると、研究上の問いは一貫して「どうすればモデルが環境の中で仕事を完遂できるか」に寄ってきた。 今回の製品化は、その研究トラックに enterprise-grade の connectors、permissions、evals、audit を被せたものと見るのが自然である。

2022

ReAct

推論と行動を往復させる基本形が広まり、言語モデルを単なる応答器ではなく、外部環境と相互作用する実行主体として捉える視点が定着し始めた。

2023

Toolformer / WebArena

ツール利用と realistic web task が評価対象になり、prompt quality より tool-enabled task completion の重要性が上がった。

2024

WebVoyager / SWE-bench / GAIA / OSWorld

ブラウザ、コード、PC 操作、複数ドメイン推論までベンチマークが広がり、agent は「実環境で完遂できるか」で測られるようになった。

2025

deep research / Operator / Agentforce / Agentspace

各社の公式製品が、web research、browser action、workflow execution、agent gallery、connectors を束ね始め、研究の論点が product surface に現れた。

2026

Cowork / Agent 365 / Frontier Suite

今年に入って見えてきたのは、単なる agent capability ではなく、governance 付きの execution layer を職場の標準面として売り出す段階に入ったことだ。

この根拠から読み取れること

AI の進化は「より自然な会話」から「より確実な完遂」へ焦点を移してきた。Cowork は、その完遂能力を knowledge worker 向けの product packaging に変えたものだと解釈できる。

Operating Implications

なぜ各社がこの方向へ進むのか。答えは productivity だけでなく governance にある

Chat は意思決定の補助には向くが、仕事を進めるには弱い。業務現場では、情報探索、ドラフト作成、更新、送信、記録、handoff が一続きになっている。 そのため各社は、agent をアプリ内へ埋め込み、社内データへ接続し、承認と監視を付けた execution layer に寄せている。

1. chat だけでは成果物まで届きにくい

仕事の多くは、答えを得ることではなく、文書更新、表計算修正、会議調整、ticket 起票、channel 通知のような action で完了する。だから各社は artifact-native execution を前に出し始めた。

2. context 接続が価値の源泉になった

Work IQ、Agentspace connectors、Slack conversational data、OpenAI apps、Claude connectors が共通して示すのは、一般知識よりも「自社固有の文脈に grounded されること」が差別化要因になっていることだ。

3. 長時間実行が product requirement になった

Deep research や Cowork は、数分から数十分の multi-step task を前提にする。これは single-turn chat ではなく、background execution と progress visibility を必要とする。

4. governance なしでは enterprise へ広がらない

Anthropic は audit logging 不足を明示し、Microsoft は Agent 365 と E7 で trust を売りにしている。つまり、agent の拡張は capability 競争であると同時に control plane 競争でもある。

5. 競争軸は「誰が smartest か」から「誰が safest to deploy か」へ広がる

導入現場では、task completion、tool success、permission boundary、auditability、retention、incident response が同時に問われる。Cowork はこの複合要求に対する製品面の回答である。

6. 導入は narrow workflow から始まる

現実的には、meeting prep、weekly report、incident summary、sales brief、IT help desk など、成果物と review point が明確な業務から広がる。万能 assistant 期待のままでは運用設計が崩れやすい。

Key Takeaway

結論

Cowork は、単に新しい buzzword が増えたという話ではない。より正確には、agent が knowledge work を成果物まで進める execution layer として製品化され始めたことを示すシグナルである。 名称を明示しているのはまだ一部の企業だが、方向性そのものはすでに横展開している。今後の比較軸は、どのモデルが最も賢いかだけでなく、どの platform が context、action、approval、governance を一体で運べるかになる。
