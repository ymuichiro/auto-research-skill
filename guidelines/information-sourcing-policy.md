# 情報収集ポリシー

## 1. 目的

`Auto Research Digest` は、LLM、AI、AI Agent に関する技術トレンド、実務ユースケース、導入論点を、経営層向けに中立的に整理して発信する。公開内容の信頼性を担保するため、情報探索の段階と公開根拠の段階を明確に分ける。

## 2. 基本方針

1. トピック探索は広く行う。
2. 公開時の根拠は、公式ドキュメントまたは論文として確認できるものに限定する。
3. 公開ページに載せる出典は `publishedSources` に登録したものだけとする。
4. 出典のラベル、URL、種別、必要に応じて公開日を明示する。
5. 新規記事または大幅改稿に進む条件として、対象トピックについて公式ドキュメントまたは論文の URL を 20 件以上収集する。

## 3. ソース区分

### 採用可能

- ベンダーや標準化団体などの公式ドキュメント
- 論文として公開され、リンクと出典が確認できるもの
- 研究機関・学会・公式リポジトリに紐づく一次情報

### 探索には使えるが、そのまま公開根拠には使わない

- ニュース記事
- 個人ブログ
- SNS 投稿
- カンファレンス現地メモ
- 二次まとめ記事
- 他者が編集した要約ページや解説記事

### 原則として除外

- 出典不明の図表
- リンク切れで原典確認できない情報
- 著者、公開主体、発行時期が不明な資料
- 推測やリークに依拠した未確認情報

## 4. 公開前チェック

公開前に少なくとも以下を満たす。

1. 根拠リンクが生きている。
2. 記事本文の主要な主張が `publishedSources` の範囲で説明できる。
3. 公式ドキュメントか論文かを `type` で区別している。
4. 執筆対象の調査インベントリとして、公式ドキュメントまたは論文の URL を 20 件以上確保している。
5. ソースが複数ある場合、主張の役割分担が曖昧でない。
6. 日付が重要な場合は、本文と出典表示の両方で整合が取れている。

## 5. 公開ページでの扱い

- 公開ページには、未検証ソースや探索段階のメモを表示しない。
- `publishedSources` 以外の URL は原則として記事末尾の根拠一覧に出さない。
- 本文で広い話題を扱う場合でも、根拠として提示する部分は一次情報で裏取りできる内容に限定する。
- `publishedSources` や「公開根拠」欄には、ニュースサイト、ブログ、キュレーションメディア、転載ページなど第三者著作物の URL を含めない。
- 第三者著作物を探索に利用した場合でも、公開ページには原典の公式ドキュメントまたは論文 URL のみを掲載する。

## 6. 1記事あたりの最小基準

- 新規記事または大幅改稿を公開候補にする前に、対象トピックの公式ドキュメントまたは論文 URL を 20 件以上収集する。
- 20 件未満しか一次情報が集まっていない場合は、記事化を急がず、調査メモまたは下書きのまま保留する。
- 情報量が多いトピックでは、20 件は下限であり、記事本文はそれらを比較・整理した上で論点を再構成する。

## 7. 更新時の原則

- 既存記事を更新する場合は、根拠リンクの有効性を再確認する。
- 新しい公式文書や論文が出た場合は、旧来の主張をそのまま温存せず、更新履歴の必要性を判断する。
- 一時的な話題性よりも、検証可能性を優先する。
