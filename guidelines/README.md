# Guidelines

このディレクトリでは、`Auto Research Digest` を継続運用するための調査・編集ルールを管理します。

- `information-sourcing-policy.md`
  公開前の根拠確認、採用可能なソース、除外対象、公開ページ上の出典表示ルールを定義します。
- `editorial-expression-guideline.md`
  ニュースサイトとしての文体、トーン、中立性、図解の見せ方、避ける表現を定義します。
- `design-system.md`
  全ページへ共通適用する色、書体、余白、ナビ、カード、記事セクション、根拠一覧のデザイン原則を定義します。

運用上の原則は次の 3 点です。

1. 話題の探索は広く行ってよいが、公開時の根拠は公式ドキュメントか論文に限定する。
2. 新規記事または大幅改稿に進む前に、対象トピックの公式ドキュメントまたは論文 URL を 20 件以上収集する。
3. 配信ページでは、判断材料と推測を混同させず、公開根拠欄にニュースサイトやブログなど第三者著作物を載せない。
4. デザイン変更はページ単位で閉じず、共通テンプレートと shared class に還元する。
5. 遡及記事では、記事の日付以前に公開された一次情報だけを採用し、official にはベンダー公式の announcement / release / launch / product update を含める。
6. 通常運用は週 1 本の公開を基本とするが、週次運用でも 20 件以上の一次情報インベントリと十分な記事密度を維持する。
7. 週次で単独テーマが薄い場合は、2 から 4 個の収束したシグナルを束ねた roundup に切り替える。
