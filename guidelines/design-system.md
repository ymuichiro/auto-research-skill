# デザインシステム

## 1. 方向性

`Auto Research Digest` の公開面は、ダークトーンの編集誌面を基本とする。  
ページごとに別テーマを作るのではなく、トップ、アーカイブ、記事、根拠一覧まで同じ visual language で統一する。

基準は次の 3 点。

1. 全ページ統一
2. 情報構造は維持
3. 新しい見た目は shared class とテンプレートで吸収

## 2. 色

- 背景: 深い黒に近いダークネイビー
- 面: 背景より一段明るいサーフェス
- 罫線: 細い低コントラストのボーダー
- 強調: シアン系シグナルカラー
- 文字:
  - 主見出しは明るい白系
  - 本文は青みのあるライトグレー
  - 補助情報は暗めのグレー

原則として、配色は「本文可読性」と「誌面の静けさ」を優先し、強い警告色や多色展開は行わない。

## 3. タイポグラフィ

- 本文と見出しは `Noto Sans JP`
- ラベル、タグ、日付、ナビ、補助メタは `DM Mono`
- 見出しは太すぎず、`medium` を基準にする
- 大見出しは文字間を少し詰め、短い行長で誌面感を出す
- モノスペースのラベルは uppercase と広い letter spacing を基本にする

## 4. レイアウト原則

- ヘッダーは常に上部固定で、細いボーダーと半透明背景を使う
- メイン幅は広げすぎず、記事可読性を優先する
- ヒーローは大きな導入見出しと短い要約に絞る
- 主要セクションは dark surface 上の薄いボーダーで分ける
- 余白は大きめに取り、情報量が多くても圧迫感を避ける

## 5. コンポーネント原則

### セクション

- `section-kicker` は共通の眉ラベルとして使う
- `panel-block` と `report-section` は同じ誌面ルールの別用途として扱う
- 記事本文では semantic class を維持し、見た目は CSS 側で吸収する

### 一覧カード

- `article-card` は細いボーダー、暗い面、控えめな hover を基本とする
- カテゴリ、タグ、日付、根拠数はモノスペースのメタ表現で揃える
- 一覧ページでもボタン然とした UI より、誌面の導線として見せる

### インフォグラフィック

- `stat-card`、`comparison-card`、`timeline-item`、`callout-panel` は残す
- ただし白いカード群ではなく、誌面の中の構造化ブロックとして再設計する
- 数字やラベルは目立たせすぎず、本文の補助線として使う

### 根拠一覧

- `publishedSources` は本文より一段モノスペース寄りのメタ表現にする
- 種別、公開日、URL、Open 導線は同じ dark editorial system 上で整える
- 公開根拠の信頼性が伝わることを優先し、装飾的なカード化は避ける

## 6. 実装ルール

- ページ個別の inline style は追加しない
- 新しい見た目が必要な場合は、まず `src/styles/input.css` に shared class を追加する
- 共通シェルの変更は `scripts/lib/render.mjs` で行う
- 記事ソースの `meta.json`、`body.ja.html`、`body.en.html` の契約は変えない

## 7. 新規ブロック追加時の判断

新しいブロックを追加する場合は、次の順序で判断する。

1. 既存 class で吸収できないか
2. 類似ブロックへ統合できないか
3. どうしても必要なら shared class を追加する

「このページだけ特別」は原則として作らない。  
新しい見た目を入れる場合も、トップ、アーカイブ、記事で再利用可能な形へ一般化する。
