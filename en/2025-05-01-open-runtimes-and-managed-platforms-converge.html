What Stood Out

The main signals visible at that point

Google is pushing protocol and platform together

Google's multi-system agent work and A2A announcement assume coordination that does not stop at a single vendor boundary. The notable move is to launch hosted infrastructure while also promoting an external interoperability protocol.

Anthropic and Microsoft are thickening the developer runtime layer

Claude Code and Semantic Kernel Agents GA both reinforce the idea that agents are part of a broader development loop and worker topology, not just a prompt template. The relationship between SDK and host environment becomes more important.

Combining open and managed layers becomes the practical path

By this point, the choice is no longer between fully self-hosted and fully managed. A mixed approach is emerging: keep protocols or SDKs open where possible, while using managed features for operations.

Use Cases

Use cases that already looked practical at that time

Multi-tool coding loops

  • Split code exploration, summarization, test execution, and review preparation across multiple tools or workers.
  • Even with role-specific workers, keeping the final code-application step with a human remains the safer pattern.

Enterprise assistants that cross several systems

  • Connect search, ticketing, notifications, and approval systems to propose responses and next actions.
  • Protocol and SDK layers like A2A can make future system changes easier to absorb.

Operating Implications

What had to be decided first in design, evaluation, and operations

Observed implication

By May, the core design question was less about feature checklists and more about which layers should stay open versus managed.

  • Decide which parts of protocol, tool schema, and session management should be standardized internally.
  • Separate workflows that fit one agent from those that really benefit from role-based workers.
  • Avoid coupling too tightly to one vendor if more endpoints or providers are likely later.

Key Takeaway

The provisional conclusion that could already be drawn

By May 2025, the agent stack looked less like a single-product decision and more like a design problem about how to combine connection protocols with operating infrastructure.
